Feature Layer(要素图层)是一种特殊的GraphicsLayer，它继承GraphicsLayer，用来
对服务图层中的要素服务进行显示，同时还提供了支持表达式过滤，要素的关联查询以及
在线编辑等功能。

用于对要素服务实现查询，渲染，编辑等操作。
通过设置FeatureLayer的setDefinitionExpression属性，还可以实现对数据的过滤。

要素图层有不同的查询显示模式：快照模式，选择模式，按需模式

·快照模式：就是将地图范围内的数据一次加载进来，当再次移动地图，范围改变的时候，
不再向服务起请求；

·选择模式：需要进行一个选择的操作，用矩形工具选择一个范围，就可以看到这个范围的
要素已经加载进来了，这个选择与查询可是有区别的。

·按需模式：指的是随着地图范围的改变要素才加载，按照所需的地图范围向服务起请求
所要显示的要素。

定义显示模式
FeatureLayer fl = new FeatureLayer(url, {
  mode: FeatureLayer.MODE_ONDEMAND,
})

FeatureLayer.setDefinitionExpression('name like ''')


影像服务

ArcGIS服务使得可以通过Web提供对栅格（及影像）和相关功能的访问，使影像能够被
web和其他客户端应用获取。包括单景影像、多景影像、作为镶嵌影像、多景影像、作为
影像目录、动态处理。

ArcGIS影像服务的数据源可以使栅格数据集（来自磁盘上的地理数据库或文件）、镶嵌
数据集或者引用栅格数据集或镶嵌数据集的图层文件

ArcGIS影像服务发布的影像服务，使用动态镶嵌和在线处理技术，客户端不仅可以快速
访问影像还能对其元数据进行查询，处理影像数据。
·快速显示（动态服务、缓存服务）
·数据输出：像素值、原始数据和处理后的数据
·动态镶嵌和影像目录
·动态影像处理：服务起端栅格函数
·影像量测
·影像服务编辑（增加、删除、更新）
·影像服务高速缓存

展现各种数据，如绘制的图形，查询返回的结果等。GraphicsLayer在客户端数据表达方面
有非常重要的作用。它可以根据各种请求动态地客户端显示一些符号化的几何对象--Graphic

动态图层服务

动态图层服务具体的表现就是图层是可以被动态作为服务发布。动态图层服务可以用于海量
图层发布与共享，特别适用于图层数量具体，并且数量增加的情况

相对于动态图层服务，那么我们以前发布的服务可以称为“静态服务”，因为这些服务是用
ArcGIS for Service提前发布好的，我们在用的时候只能使用这些发布好的服务，而
动态图层服务就比较灵活，它不需要提前发布，而是在使用的时候我们使用的时候由ArcGIS
for server 动态发布；
使用动态图层服务功能，不仅仅可以动态的发布图层，而且还可以对动态服务中的图层进行
重新排序，在发布地图的时候，有一个允许更改图层和符号的选项，只要我们将这个选项选上，
那么就可也将一个图层拖动到一个新的位置。

要使用ArcGIS FOR Server动态发布图层的这个功能，需要在server上做一个工作空间注册
的配置，通过该功能，可以给当前的地图服务动态添加数据（矢量和栅格都可以），而这些
数据是REST目录不可见的，这对保护一些数据和经常更新的数据来说是很有作用。
